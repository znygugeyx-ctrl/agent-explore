# 蛊真人小说 QA 测试计划

## 目标

评估 MSA (Memory Sparse Attention) 对长篇中文网络小说的记忆检索与问答能力。

- 小说：《蛊真人》，634 万字，2345 节，约 589 万 tokens
- 模型：MSA-4B (Qwen3-4B-Instruct 微调)
- 基础设施：4×L40S 48GB, 384GB RAM

## 测试集设计

### 规模

50 个 QA 对，分两级难度各 25 个。

### 难度定义

**Easy（单节检索）**：答案明确出现在某一节中，无需跨节推理。
- 例："方源在开窍大典上测试的资质等级是什么？" → "丙等"
- 考察：MSA 能否从 2345 节中准确检索到包含答案的那一节

**Hard（跨节推理）**：答案需要综合多节信息，或涉及隐含关系、因果链、时序比较。
- 例："影无邪为什么要偷春秋蝉？" → 需综合影无邪的动机（多节铺垫）
- 考察：MSA 的 top-k 检索能否覆盖分散在不同节中的相关信息

### 类别覆盖

| 类别 | 说明 | Easy | Hard |
|---|---|---|---|
| character | 人物身份、属性 | 6 | 3 |
| item | 蛊虫名称、能力 | 5 | 4 |
| event | 情节事件 | 5 | 5 |
| location | 地点、势力 | 4 | 3 |
| relationship | 人物关系 | 3 | 4 |
| reasoning | 因果、动机、比较 | 2 | 6 |

### 卷次覆盖

问题均匀分布于小说六卷，避免集中在开头：
- 第一卷（节 1-199）：~10 题
- 第二三卷（节 200-649）：~10 题
- 第四五卷（节 650-1968）：~15 题
- 第六卷（节 1969-2345）：~15 题

## 评估方法

### 自动评估

1. 将 50 个问题通过 `/ask` 接口发送给 MSA 服务
2. 收集模型回答、引用文档、延迟、生成轮数
3. 使用 Bedrock Claude Sonnet 4.6 作为 LLM Judge 评分（0-5 分制，与 hotpotqa 复现一致）

### 评分标准

| 分数 | 含义 |
|---|---|
| 5 | 答案与标准答案完全一致 |
| 4 | 核心信息正确，略有冗余 |
| 3 | 基本正确但有偏差或不完整 |
| 2 | 部分相关，遗漏重要信息 |
| 1 | 方向正确但事实错误 |
| 0 | 完全不相关或胡说 |

### 分析维度

- 整体 LLM Score（与 hotpotqa 的 4.172 对比）
- Easy vs Hard 分数差异（验证难度分级有效性）
- 按类别分组的分数（哪类问题 MSA 最擅长/最弱）
- 按卷次分组的分数（前期 vs 后期内容是否有差异）
- 检索命中率：引用的文档是否包含 source_section

## 预期结果

### 假设

1. **Easy 题分数显著高于 Hard 题**（验证难度分级）
2. **整体分数低于 hotpotqa (4.172)**，因为：
   - 小说是中文，模型和 prompt 模板是英文优化的
   - 小说每节 ~3400 字远大于 hotpotqa 的短文档
   - 小说的叙事风格比百科文档更隐晦
3. **character/item 类别分数高于 reasoning 类别**
4. **后期卷次分数可能略低**（文档数量大，检索难度增加）

### 成功标准

- Easy 平均 LLM Score ≥ 3.0
- Hard 平均 LLM Score ≥ 1.5
- 检索命中率（引用文档包含正确节）≥ 60%
