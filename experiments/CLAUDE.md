# experiments - Hypothesis Validation

## Purpose

Each subdirectory is a self-contained experiment validating a specific hypothesis.

## Directory Convention

```
experiments/
  001_mask_vs_remove_tools/
    hypothesis.md      # What we're testing and why
    config.yaml        # All experiment parameters
    run.py             # Entry point (self-contained)
    results/           # Output data
      run_1.json
      run_2.json
      summary.md       # Conclusions with data
```

## Requirements

- **Reproducible**: same config.yaml + run.py must produce comparable results
- **Data-supported**: conclusions backed by quantitative results
- **Self-contained**: each experiment has its own config, no shared mutable state
- **Documented**: hypothesis.md explains what and why before running

## Naming

Use `<NNN>_<short_name>/` format. Number sequentially.

## Analysis Convention

After collecting results, experiments follow a three-file pattern:

```
results/
  *.json / *.jsonl    # Raw output from run.py
  analysis.md         # Human-written conclusions with key numbers and interpretation
  report.md           # Generated by analyze.py — full data tables, per-task breakdown
```

- **`analyze.py`** (in experiment root): standalone script, reads `results/**/*.json`, writes `results/report.md`. Run with `python3 experiments/<NNN>_*/analyze.py`.
- **`report.md`**: generated, data-dense, typically in Chinese. Contains strategy comparison tables, per-subgroup breakdowns, raw numbers.
- **`analysis.md`**: authored summary. States which hypotheses were confirmed/rejected, key findings, and implications for future experiments.

Not every experiment needs `analyze.py` — simple experiments may write `analysis.md` directly from run output.
